# MinIO

官网地址：https://min.io/

Minio 需要使用到：服务端口（默认为 9000）和控制台端口（默认为 9001）

- 服务端口

    默认：9000

    用于提供 Minio 对象存储服务。浏览器或者客户端可以通过该端口上传、下载和管理存储桶中的对象数据。

- 控制台端口

    默认：9001

    用于提供 Minio 的 Web 管理控制台服务。浏览器访问该端口就是访问 MinIO 控制台。

# docker 安装 MinIO

步骤1：拉取 minio 镜像

```shell
docker pull minio/minio
```

步骤2：宿主机创建 minio 配置及数据目录

```shell
mkdir -p /data/minio/config
mkdir -p /data/minio/data
```

步骤3：docker 启动 minio

> 执行下述命令前，检查宿主机映射的端口号是否被占用。
>
> 方式1：使用 netstat 命令
>
> ```shell
> netstat -tuln | grep <端口号>
> ```
>
> 方式2：使用 lsof 命令
>
> ```shell
> lsof -i :<端口号>
> ```
>
> 方式3：使用 ss 命令：
>
> ```shell
> ss -tuln | grep <端口号>
> ```

下述命令，将宿主机的 9500 端口映射到 9000，9501 d端口映射到 90001。

```shell
docker run \
-d --restart=always \
--name minio \
--network=host \
-p 9500:9000 \
-p 9501:9001 \
-e "MINIO_ACCESS_KEY=minioadmin" \
-e "MINIO_SECRET_KEY=minioadmin" \
-v /data/minio/data:/data \
-v /data/minio/config:/root/.minio \
minio/minio server /data \
--address ":9500" --console-address ":9501"
```

上述：`--address` 来指定 Minio 服务的服务监听端口，`--console-address` 来指定 Minio 控制台的监听端口。

步骤4：验证 minio 服务

假设宿主机的IP为：192.168.100.100，浏览器访问 minio 服务：http://192.168.100.100:9521
